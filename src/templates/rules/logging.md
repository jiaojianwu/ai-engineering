# 对话日志记录规范

## 日志格式

每次对话结束后，AI 应输出以下格式的日志：

```
## 对话摘要

**任务类型**：[快速流程/标准流程/完整流程]
**任务描述**：[简要描述任务内容]
**涉及文件**：[列出所有读取/修改的文件]
**执行步骤**：
1. [步骤1]
2. [步骤2]
...

**结果**：[成功/失败]
**耗时**：[预估时间]
**备注**：[如有]
```

## 日志级别

| 级别 | 说明 | 使用场景 |
|------|------|----------|
| INFO | 一般信息 | 正常操作记录 |
| WARN | 警告信息 | 潜在问题或需要关注 |
| ERROR | 错误信息 | 操作失败或异常 |

## 日志输出位置

- 默认输出到控制台
- 可配置输出到文件（通过环境变量 `AI_LOG_FILE`）

## 敏感信息处理

- 不记录用户的敏感信息（密码、密钥等）
- 不记录完整的 API 响应内容
- 只记录必要的元数据

## 日志示例

```
## 对话摘要

**任务类型**：标准流程
**任务描述**：为 TagManage 页面添加批量删除功能
**涉及文件**：
- src/pages/TagManage/index.tsx (读取)
- src/pages/TagManage/docs.md (读取)
- src/services/tagService.ts (读取)
- src/pages/TagManage/index.tsx (修改)

**执行步骤**：
1. 读取 checklist.md 确认检查流程
2. 读取 file-priority.md 确认文件优先级
3. 读取 TagManage/docs.md 了解业务需求
4. 读取 TagManage/index.tsx 了解现有实现
5. 读取 tagService.ts 了解 API 接口
6. 实现批量删除功能
7. 运行 tsc 检查
8. 运行 lint 检查

**结果**：成功
**耗时**：约 45 分钟
**备注**：新增了 batchDeleteTags API 接口
```
